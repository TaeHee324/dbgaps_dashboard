# PRD — DBGAPS 포트폴리오 자동화 대시보드

## 목표

DB GAPS 대회 기간 동안 팀의 의사결정, 리스크 점검, 보고서 작성 부담을 줄이는 ETF 포트폴리오 자동화 시스템을 만든다.

## 사용자

DB GAPS 대회 팀원 (3~5명). 코딩보다는 투자 판단이 주 업무이며, 도구는 CLI 또는 웹 브라우저로 사용한다.

## 핵심 기능 (1차 — 완료)

- ETF 마스터 파일 관리 (188개, raw_ticker/code/name/risk_type/asset_class)
- ETF 일별 종가 저장 및 증분 업데이트 (pykrx)
- 단일 포트폴리오 백테스트 (드리프트 / 주기 리밸런싱)
- 성과지표 계산: CAGR, MDD, Alpha, Beta, 연간변동성, 승률, 샤프지수, 칼마지수
- 벤치마크 대비 성과 비교
- 매매일지 기반 현재 보유 평가 (평가금액, 비중, 미실현손익)
- 대회 규칙 체크: 개별 ETF 20% 상한, 위험자산 70% 상한
- 회전율 체크: 초기 누적 80%, 주간/월간 10%

## 핵심 기능 (2차 — 진행 예정)

- 테스트 골격 및 smoke test (phase-0)
- 실제 ETF 가격 데이터 수집 (phase-1)
- 누적수익률, MDD, 월별수익률 그래프 (phase-2)
- 월간보고서 Markdown 자동 생성 (phase-3)

## 핵심 기능 (3차 — 기술스택 결정 후)

- 웹 대시보드 (Railway 배포, 기술스택 TBD)
- 성과지표 카드, 그래프 표시
- 현재 포트폴리오 비중, 규칙 체크 결과 표시

## 핵심 기능 (4차 — 후순위)

- 월간보고서용 시황 자료 수집 자동화
- 주요 시장지표 수집 (KOSPI, S&P500, 환율, 금리, 금)

## MVP 제외 사항

- 실시간 장중 시세 (증권사 OpenAPI 미확보)
- 웹에서 매매일지 직접 수정
- 실제 주문/매매 연동
- AI 투자 판단 추천
- 외부 공개용 웹서비스

## 디자인 방향

- 내부 팀 도구이므로 UI 미적 완성도보다 데이터 정확성과 속도 우선
- 그래프는 matplotlib 정적 PNG로 먼저 구현
- 웹은 읽기 전용 대시보드로 시작
