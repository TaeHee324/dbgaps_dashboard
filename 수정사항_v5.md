### 1-2. 남아 있는 버그 2개 (미수정)

#### Bug A — `actual_nav()` 현금 누락 (`api/routers/dashboard.py:547`)

```python
# 현재 (잘못됨)
total_value = portfolio_value           # ETF 시가만, 현금 제외
cumulative_return = (total_value - INITIAL_CAPITAL) / INITIAL_CAPITAL

# 올바른 코드
total_value = portfolio_value + cash    # ETF 시가 + 현금
cumulative_return = (total_value - INITIAL_CAPITAL) / INITIAL_CAPITAL
```

`daily_return` 계산도 동일하게 수정해야 한다:
```python
# 현재 (잘못됨)
daily_return = (portfolio_value - prev_value) / prev_value

# 올바른 코드 — prev_total_value도 cash 포함 값으로 추적
daily_return = (total_value - prev_total_value) / prev_total_value
```

**영향**: 현재 포트폴리오는 현금 73,985원 / 10억 = 0.007%로 오차가 미미하다.  
그러나 이후 매도 시 현금이 늘면 daily_return이 매도 당일에 잘못 계산된다.  
(종목 매도 → ETF 시가 감소 → prev_value 대비 폭락처럼 보임)

#### Bug B — `actual_nav()` 가격 누락 시 NAV 폭락 (`dashboard.py:533~536`)

```python
# 현재: 가격 없으면 해당 ETF가 0으로 계산됨
if dt in price_pivot.index and code in price_pivot.columns:
    close = price_pivot.at[dt, code]
    if not pd.isna(close):
        portfolio_value += qty * close
# 가격 없는 날 → portfolio_value 급감 → 가짜 drawdown 발생
```

**수정 방법**: price_pivot에 ffill 적용
```python
price_pivot = prices_df.pivot_table(...).ffill()  # 전날 종가로 대체
```

**영향**: 국경일·서킷브레이커 등 특정 ETF만 거래 중단되면 가짜 급락이 나타난다.

### 1-3. 구조적 문제 — KPI 성격 혼재

현재 `computeMetricsFromNav()`는 실제 운용 24일 데이터로 CAGR·샤프·칼마 등을 계산하는데, 이는 통계적으로 의미 없다.

| 지표 | 24일 데이터로 계산하면 |
|------|----------------------|
| CAGR | 단기 수익을 연환산 → 실제 72%+ 표시 (왜곡) |
| 샤프 | 표준오차 ≈ 1/√24 ≈ 20% → 신뢰구간 너무 넓음 |
| 칼마 | CAGR 기반이므로 동일하게 왜곡 |
| 소르티노 | 하방 수익률 수가 너무 적음 |
| 월별승률 | 현재 1개월 → 100% 또는 0% |
| VaR 95% | 24개 중 5th percentile = 1~2개 → 무의미 |

## 2. KPI 분류 체계 (최종)

### 카테고리 1 — 실제 운용 성과
> 데이터 출처: `actual_nav` (trade_log DB 기반)  
> 기간: 2026-05-04(첫 매수일) ~ 현재  
> 성격: 실제 벌고 잃은 것의 기록. 시간이 쌓일수록 의미 증가.

| KPI | 공식 | 최소 데이터 |
|-----|------|------------|
| 누적수익률 | `(total_value - 초기자본) / 초기자본` | 1일 |
| MDD | `min(drawdown)` | 1일 |
| 일간승률 | `daily_return > 0 인 날 / 전체 거래일` | 5일 이상 권장 |
| 연환산 변동성 | `std(daily_return) × √252` | 10일 이상 권장 |
| MDD기간 | 고점 → 저점(또는 회복) 캘린더 일수 | 1일 |

### 카테고리 2 — 전략 특성 (백테스트)
> 데이터 출처: `backtest_nav` (현재 포트폴리오 고정비중, 과거 가격 적용)  
> 기간: 사용자 선택 (1Y / 6M / 3M)  
> 성격: "이 전략이 역사적으로 어떤 특성을 가졌는가"  
> **주의**: 종목 변경 시 이 수치는 재계산된다 (설계 의도)

| KPI | 최소 권장 기간 | 현재 가용 데이터 |
|-----|--------------|----------------|
| CAGR | 60거래일 이상 | 3M=58일 / 6M=118일 / 1Y=242일 |
| 샤프 | 100거래일 이상 | 6M 이상 권장 |
| 칼마 | CAGR와 동일 | 3M부터 표시 |
| 소르티노 | 50거래일 이상 | 3M부터 표시 |
| 월별승률 | 3개월 이상 | 3M=3개월 / 6M=6개월 / 1Y=12개월 |
| VaR 95% | 20거래일 이상 | 3M부터 표시 |

## 4. 구현 계획

### Step 1 — Backend: `actual_nav()` 버그 2개 수정
**파일**: `api/routers/dashboard.py`

**수정 A — 현금 포함 total_value 및 daily_return**
```python
# prev_total_value를 별도 추적
prev_total_value: float | None = None

for dt in all_dates:
    # ... 거래 반영 ...
    portfolio_value = ...  # ETF 시가 합계
    cash = INITIAL_CAPITAL - cumulative_buy_cost + cumulative_sell_proceeds
    total_value = portfolio_value + cash  # ← 수정

    daily_return = 0.0
    if prev_total_value is not None and prev_total_value > 0:
        daily_return = (total_value - prev_total_value) / prev_total_value  # ← 수정

    cumulative_return = (total_value - INITIAL_CAPITAL) / INITIAL_CAPITAL  # ← 수정

    # ... nav_list.append(...) ...
    prev_total_value = total_value  # ← 수정 (prev_value 대신)
```

**수정 B — 가격 forward-fill**
```python
price_pivot = prices_df.pivot_table(
    index="date", columns="code", values="close", aggfunc="last"
).ffill()  # ← 추가: 가격 공백 있는 날 → 전날 종가 사용
```

**검증**: `python src/run_engine.py` 실행 불필요, API 서버 재시작만으로 반영됨

### Step 3 — Frontend: `page.tsx` 전략 섹션 추가
**파일**: `frontend/app/page.tsx`

```typescript
// 추가할 상태
const [strategyPeriod, setStrategyPeriod] = useState<"1Y" | "6M" | "3M">("1Y");
const backtestNavQuery = useBacktestNav();  // 기존 훅 재사용

// 기간별 필터링 (거래일 기준)
const PERIOD_DAYS = { "1Y": 252, "6M": 126, "3M": 63 };
const strategyPoints = useMemo(() => {
  const pts = backtestNavQuery.data ?? [];
  const days = PERIOD_DAYS[strategyPeriod];
  return pts.slice(-days);

// 전략 지표 계산
const strategyMetrics = useMemo(
  () => computeStrategyMetrics(strategyPoints),
  [strategyPoints]
);

// 실제 운용 지표 계산 (기존 computeMetricsFromNav 대체 예정)
const actualOpsMetrics = useMemo(
  () => computeActualOpsMetrics(actualNavQuery.data ?? []),
  [actualNavQuery.data]
);
```

## 5. 충돌·부작용 검토

| 위험 | 내용 | 대응 |
|------|------|------|
| `actual_nav` 리팩토링 영향 범위 | `daily_return` 값이 바뀌면 NAV 차트·Drawdown도 미세하게 달라짐 | Bug A 수정 후 시각적으로 확인 |
| `backtest_nav` 가중치 변경 문제 | 종목 사고팔면 전략 특성 수치가 소급 재계산됨 | 전략 스트립에 "현재 포트폴리오 기준" 레이블로 의도 명시 |
| `PortfolioSummary` 타입 변경 영향 | `alpha`, `beta`, `sharpe`, `calmar` nullable로 이미 수정됨 → portfolio 페이지 `fmt()` 함수도 수정 완료 | tsc 에러 없음 확인됨 |
| `useBacktestNav()` 중복 fetch | 전략 스트립 추가 시 `backtestNavQuery`와 비교 탭이 동일 훅 사용 → TanStack Query 자동 캐시 공유 | 별도 조치 불필요 |
| 3M 기간 선택 시 데이터 부족 | 3M=58거래일, 샤프·소르티노는 통계적으로 불안함 | tooltip으로 명시, 수치는 표시 |
| 비교 탭 (`/comparison`) | `backtest_nav`와 `comparison_nav` 계속 사용 중 — 이번 변경 무관 | 영향 없음 |
| `monthly_returns.csv` | 운용현황 페이지에서 더 이상 사용 안 함. 비교 탭에서도 사용 여부 미확인 | 삭제 전 확인 필요 |
