그룹 A의 NAV/Drawdown 차트가 기존 backtest 시뮬레이션 대신 **실제 거래 기반 NAV**를 사용하도록 변경됨.
이를 위해 그룹 B가 `/api/actual-nav` 신규 엔드포인트를 먼저 구현해야 그룹 A가 이를 참조 가능.
→ 그룹 A는 그룹 B 완료 후 실행.

## 그룹 A — 운용 대시보드 페이지 전면 재구성 (Frontend, B 완료 후 실행)

**수정 파일:** `frontend/app/operations/page.tsx`, 관련 컴포넌트  
**선행 조건:** 그룹 B의 `/api/actual-nav` 엔드포인트 구현 완료

### A1. NAV / MDD 차트 — 실제 거래 기반으로 전환
- 기존 `useBacktestNav`(시뮬레이션) → 그룹 B가 새로 만든 `useActualNav`(`/api/actual-nav`) 훅으로 교체
- **이유:** 총 자산 현황 카드(실제 holdings 기반)와 NAV 차트(백테스트 시뮬레이션)가 다른 데이터를 보여주면 말단값이 불일치하여 혼란 발생. 두 카드가 동일한 "실제 포트폴리오 가치 시계열"을 공유해야 함.
- NAV 차트 끝점 = 총 자산 현황 카드 현재값으로 일치
- 총 자산 현황 카드는 `/api/actual-nav` 시계열의 마지막 포인트 사용

### A2. 상단 KPI 카드 업데이트 (KpiStrip)
- 포트폴리오명("현재 운용 포트폴리오") 표시
- **실제 데이터 기간 표시:** `navQuery.data[0].date ~ navQuery.data[마지막].date` 값을 직접 읽어 렌더링 (하드코딩 금지)
- **현재 포트폴리오 스펙 카드 삭제** (KPI 카드가 동일 역할이므로 중복 제거)

### A3. 총 자산 현황 카드 신규 추가
- 10억 기준: 현재 평가금액 / 평가손익 / 현금 비중을 파이차트 + 수치로 표시
- 데이터 출처: `/api/actual-nav` 시계열의 마지막 포인트 (그룹 B 구현)
- 현금 = 10억 - 현재 평가금액

### A4. 일별 수익률 히트맵으로 교체
- 기존 `MonthlyBarChart`(좌측 월별 바차트) 완전 삭제
- 히트맵이 카드 전체를 꽉 채우도록 재구성
- 오늘 날짜 기준 현재 월(5월) 자동 표시

### A5. 카드 순서 정리 (최종 확정)
```
1. KPI 카드          ← 기존 핵심 지표 (실제 기간 표시 포함)
2. 총 자산 현황 카드  ← 신규 (파이차트)
3. NAV / MDD 카드    ← 실제 거래 기반으로 전환
4. 일별 수익률 히트맵 ← 월별 바차트 대체
5. 투자 규칙 카드     ← 기존 위치에서 앞으로 이동
6. 현재 보유 종목 카드 ← 기존 포트폴리오 구성
7. 회전율 카드        ← 맨 아래로 이동
```

## 그룹 B — 백엔드: 실제 NAV 엔드포인트 신설 + 회전율 & 투자규칙 수정

**수정 파일:** `src/rules.py`, `src/turnover.py`, `api/routers/dashboard.py`, `api/schemas.py`

### B0. 실제 거래 기반 NAV 엔드포인트 신설 (그룹 A 선행 요건)
- `GET /api/actual-nav` 신규 엔드포인트 추가
- 로직: `data/trade_log.json` 매매 이력 + `data/prices_daily.csv` 실제 가격 → 일별 포트폴리오 총 가치 계산
- 반환: `list[NavPoint]` (date, portfolio_value, daily_return, cumulative_return, drawdown)
- 총 자산 카드용 최신값 포함: 마지막 포인트의 `portfolio_value`로 현재 평가금액 파악 가능
- 기존 `/api/backtest-nav`는 유지 (포트폴리오 비교용)

### B1. 회전율 계산 수정
- 대회 룰 기준 공식 확인: `회전율 = (총 매매금액 / 평균 자산) × 100%`, `평균 자산 = (기초자산 + 기말자산) / 2`
- 초기 누적(80% 기준): 2026-06-01 ~ 2026-06-08 기간 내 누적 거래금액 기반 정확히 계산되는지 검증
- 주간(10% 기준): 오늘 날짜 기준 현재 주 월~금 범위 정확히 인식하는지 검증
- 월간(10% 기준): 오늘 날짜 기준 현재 월 1일~말일 범위 정확히 인식하는지 검증
- 오류 발견 시 수정, 타임스케일 경계 케이스(월초, 주초 등) 반복 확인

### B2. 투자규칙 계산 오류 수정
- 현재 보유 비중 계산 버그 수정 (현재 "모두 위반"으로 나오는 오류)
- 반영해야 할 규칙 전체:
  - 개별 ETF 20% 상한
  - 위험자산 총합 70% 상한
  - 세부 자산군별 상한 (국내주식_지수 30%, 국내주식_섹터 15%, 해외주식_지수 30%, 해외주식_섹터 10%, FX 및 원자재 20%, 국내채권_종합 50%, 국내채권_회사채 30%, 해외채권_종합 50%, 해외채권_회사채 30%, 금리연계형/초단기채권 50%)
- 각 ETF의 `asset_class`, `risk_type` 분류가 규칙에 올바르게 매핑되는지 확인

## 그룹 C — 포트폴리오 비교 탭 전면 개선

### C2. 포트폴리오 삭제 기능
- 비교 지표 테이블 각 행 끝에 삭제 버튼 추가
- DB에서 해당 포트폴리오 완전 삭제 (`DELETE /api/portfolios/{name}` 엔드포인트 추가 또는 활용)

### C3. 그룹핑 + 접기/펼치기
- DB PostgreSQL 포트폴리오 테이블에 `group_name` 컬럼 추가 (nullable, 없으면 "기타" 그룹으로 표시)
- UI: 그룹별 헤더 + 접기/펼치기 버튼
- 그룹 없는 포트폴리오는 "기타"로 자동 묶음

- 차트도 동일 기간으로 잘라서 표시 (기존 동작과 통일)

### C5. 지표 정렬 기능
