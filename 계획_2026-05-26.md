## 그룹 A — 운용 대시보드 페이지 전면 재구성 (Frontend, B 완료 후 실행)

**수정 파일:** `frontend/app/page.tsx`, 관련 컴포넌트  
**선행 조건:** 그룹 B의 `/api/actual-nav`, `/api/live-rules` 엔드포인트 구현 완료

### A1. NAV / MDD 차트 — 실제 거래 기반으로 전환
- 기존 `useBacktestNav`(시뮬레이션) → `useActualNav`(`/api/actual-nav`) 훅으로 교체
- **`useActualNav` 훅 신규 생성:** `frontend/lib/hooks/dashboard.ts`에 추가 (그룹 B 완료 후)
- NAV 차트 끝점 = 총 자산 현황 카드 현재값으로 일치

### A2. 상단 KPI 카드 업데이트 (KpiStrip)
- 포트폴리오명("현재 운용 포트폴리오") 표시
- **실제 데이터 기간 표시:** `navQuery.data[0].date ~ navQuery.data[마지막].date` 값을 직접 읽어 렌더링 (하드코딩 금지)
- **`LivePortfolioSpec` 카드 삭제** (KPI 카드와 역할 중복)

### A3. 총 자산 현황 카드 신규 추가
- 파이차트 + 수치로 현재 평가금액 / 평가손익 / 현금 비중 표시
- 데이터 출처: `/api/actual-nav` 시계열의 마지막 포인트의 `cash` 필드
- **현금 계산 공식 (B0에서 구현):**
  - 현금 = 10억 − 누적 매수원금 + 매도 회수금
  - ※ "10억 − 현재 평가금액"이 아님 — 평가금액이 오르면 현금이 음수가 되므로 부정확

### A4. 일별 수익률 히트맵 — 위치 이동
- `DailyHeatmap` 컴포넌트는 이미 존재 — **삭제·재구현 없이 카드 순서 4번 위치로 이동**
- 오늘 날짜 기준 현재 월 자동 표시 (기존 동작 유지)

### A5. 투자 규칙 카드 — 데이터 소스 교체
- 기존 `useRules()`(`/api/rules`, CSV 기반) → `useLiveRules()`(`/api/live-rules`, 실시간 기반)로 교체
- **`useLiveRules` 훅 신규 생성:** `frontend/lib/hooks/dashboard.ts`에 추가

### A6. 카드 순서 정리 (최종 확정)
```
1. KPI 카드
2. 총 자산 현황 카드   ← 신규 (파이차트 + 수치)
3. NAV / MDD 카드     ← 실제 거래 기반으로 전환
4. 일별 수익률 히트맵  ← 위치만 이동 (기존 컴포넌트 재사용)
5. 투자 규칙 카드      ← /api/live-rules로 교체
6. 현재 보유 종목 카드
7. 회전율 카드
```

삭제 대상: `LivePortfolioSpec`, `HoldingsCompositionPanel` 카드 (중복 또는 불필요)

## 그룹 B — 백엔드: 실제 NAV + 실시간 규칙 엔드포인트 신설 + 회전율 수정

**수정 파일:** `src/turnover.py`, `api/routers/dashboard.py`, `api/schemas.py`

### B0. 실제 거래 기반 NAV 엔드포인트 신설 (그룹 A 선행 요건)
- `GET /api/actual-nav` 신규 엔드포인트 추가 (`dashboard.py` 내 구현)
- **데이터 소스: PostgreSQL DB의 `trade_log` 테이블 + `data/prices_daily.csv`**
  - `data/trade_log.json`은 사용하지 않음 (구버전 경로 — 현재 시스템은 DB 기반)
  - `_calc_live_holdings()`와 동일하게 DB에서 직접 읽는 방식 사용
- **로직:** 거래일별 보유 상태 누적 계산 → 해당일 종가 × 수량 합산 → 일별 포트폴리오 가치 시계열
- **반환 스키마:** 기존 `NavPoint`(schemas.py에 이미 존재)에 `cash` 필드 추가
  - `date`, `portfolio_value`, `daily_return`, `cumulative_return`, `drawdown`, `cash`
  - `cash = 10억 − 누적 매수원금 + 매도 회수금` (각 포인트 시점 기준 누적 계산)
- 기존 `/api/backtest-nav`는 유지 (포트폴리오 비교용)

### B1. 실시간 규칙 체크 엔드포인트 신설 (그룹 A 선행 요건)
- `GET /api/live-rules` 신규 엔드포인트 추가 (`dashboard.py` 내 구현)
- **CRITICAL-1 준수:** `src/rules.py` import 없이 `dashboard.py` 내에서 비중 체크 직접 구현
- **데이터 소스:** `_calc_live_holdings()` 결과(실시간 live holdings) 기반
- **체크 항목:**
  - 개별 ETF 20% 상한
  - 위험자산 총합 70% 상한
  - 세부 자산군별 상한 (국내주식_지수 30%, 국내주식_섹터 15%, 해외주식_지수 30%, 해외주식_섹터 10%, FX 및 원자재 20%, 국내채권_종합 50%, 국내채권_회사채 30%, 해외채권_종합 50%, 해외채권_회사채 30%, 금리연계형/초단기채권 50%)
  - 각 ETF의 `asset_class`, `risk_type`은 `etf_master.csv`에서 읽음 (이미 `_calc_live_holdings()` 내 로직 참고)
- 반환 스키마: 기존 `RulesResponse` 재사용
- 기존 `/api/rules`(CSV 기반)는 유지 (하위 호환)
- ※ `src/rules.py`의 엔진용 버그는 이번 범위 제외 — 운용현황 페이지는 `/api/live-rules`로 해결

### B2. 회전율 계산 수정
- 대회 룰 기준 공식: `회전율 = (총 매매금액 / 평균 자산) × 100%`, `평균 자산 = (기초자산 + 기말자산) / 2`
- 검증 항목:
  - 초기 누적(80% 기준): 2026-06-01 ~ 2026-06-08 기간 내 누적 거래금액 기반 정확히 계산되는지
  - 주간(10% 기준): 오늘 날짜 기준 현재 주 월~금 범위 정확히 인식하는지
  - 월간(10% 기준): 오늘 날짜 기준 현재 월 1일~말일 범위 정확히 인식하는지
- 오류 발견 시 `src/turnover.py` 수정, 타임스케일 경계 케이스(월초, 주초 등) 반복 확인
- ⚠️ 엔진 수정 후 `python src/run_engine.py` 재실행 + `output/` 커밋 필수 (SYNC-3)

## 그룹 C — 포트폴리오 비교 탭 전면 개선

**수정 파일:** `frontend/app/comparison/page.tsx`, `api/routers/portfolios.py`, `api/schemas.py`

### C1. 현재 운용 포트폴리오 고정 표시
- 비교 탭 최상단에 현재 보유종목 기반 포트폴리오를 자동 포함 (삭제 불가 고정 행)

### C2. 포트폴리오 삭제 기능
- 비교 지표 테이블 각 행 끝에 삭제 버튼 추가
- **백엔드 `DELETE /api/portfolios/{name}` 엔드포인트는 이미 존재** — UI 버튼 연결만 구현

### C3. 그룹핑 + 접기/펼치기
- **DB 마이그레이션 선행 필요 (Railway PostgreSQL 직접 실행):**
  ```sql
  ALTER TABLE portfolios ADD COLUMN group_name TEXT;
  ```
- 마이그레이션 후 코드 수정:
  1. `api/schemas.py` — `Portfolio`, `PortfolioUpsertRequest`, `PortfolioUpsertResponse`에 `group_name: str | None = None` 추가
  2. `api/routers/portfolios.py` — 조회/저장/수정 쿼리에 `group_name` 포함
- UI: 그룹별 헤더 + 접기/펼치기 버튼, 그룹 없는 포트폴리오는 "기타"로 자동 묶음

### C4. 동일 시계열 강제 재계산 — 프론트엔드 계산 방식
- 기간 필터: `1M`, `3M`, `6M`, `1Y`, `전체`
- **구현 방식: 백엔드 재계산 없음 (CRITICAL-1 준수)**
  - `/api/comparison/nav`로 이미 전체 nav 시계열을 수신 중
  - 프론트엔드에서 선택 기간으로 nav 슬라이싱 후 CAGR/MDD/Sharpe/Calmar 직접 계산
  - `api/routers/dashboard.py`에 `start_date`/`end_date` 파라미터 추가하지 않음
- 프론트엔드 계산 공식 (JS):
  - `CAGR = (final/initial)^(365/days) − 1`
  - `MDD = max(1 − value/running_peak)` over period
  - `Sharpe = mean(daily_return) / std(daily_return) * sqrt(252)`
  - `Calmar = CAGR / abs(MDD)`

### C5. 지표 정렬 기능
- 테이블 헤더(CAGR, MDD, 샤프, 칼마) 클릭 시 오름/내림차순 토글

### C6. y축 범위 개선
- NAV, MDD 차트 y축 여백 확대하여 변화가 눈에 잘 띄도록 조정

## 보류/추후 논의 사항

- **리서치 탭 내용:** 어떤 데이터를 표시할지 미정. 플레이스홀더로 유지.
- **`src/rules.py` 엔진 버그:** 운용현황 페이지는 `/api/live-rules`로 해결되므로 이번 범위 제외. 엔진 출력 기반 규칙 체크 정확도는 추후 별도 수정.
- **홈(`/`) 리다이렉트:** `app/operations/page.tsx`는 `redirect("/")` 유지. `app/page.tsx`가 운용 대시보드 실체.
